### I. Data Format for Prediction Model:  

Format for ECG prototype output recordings:  
Data format should be about 250 samples per second (250 hertz), using + and - floating point numbers, precision of float 16 or float 32 is good enough, in units of micro volts or milli volts (to be set later), and in one of the following file formats:  
 * common Python data formats:  .csv, .txt, .mat, .h5 or .hdf5, .py, .npy or .npz
 * or spreadsheet formats: Excel or Google Sheets
   
Recording duration per session should be 30 seconds to 5 minutes during evaluation phase. Each recording should also generate a header file (.hea) with user and date identifying information. V2, V5/V6, Limb I, Limb II should be identified and be shown in rows. Timed microvolt data series will be in columns. You may use padding zeros in the beginning or end of each stream if that helps you. You can also add identifiers at the beginning or end of each stream with fixed length.   

Sample data, 10 records of PTB-XL, my Github repo:
